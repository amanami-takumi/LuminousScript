# アセット外部化と読み込み経路

## 目的

`luminas_script.py` の生成物を、従来の「単一HTMLにBase64埋め込み」方式から、以下の構成へ変更した。

- `output/CSVname_YYYYMMDDHHmmSS/`
- `output/CSVname_YYYYMMDDHHmmSS/game.html`
- `output/CSVname_YYYYMMDDHHmmSS/assets/`

生成処理の入口は [luminas_script.py](/home/web/luminous/LuminousScript/luminas_script.py#L772)。

## 出力構成

生成時に `generate_html()` が出力先ディレクトリを作成し、その中に `game.html` と `assets/` を出す。

- `game.html`
  - シナリオJSON
  - 設定JSON
  - `元アセット名 -> 読み込みメタ情報` のマップ
- `assets/*.bin`
  - 難読化済みの画像・音声・favicon実体
- `assets/_local/*.bin.js`
  - `file://` 直開き時専用のローカル補助スクリプト

アセット収集は [luminas_script.py](/home/web/luminous/LuminousScript/luminas_script.py#L647)。

## 難読化の内容

難読化は2段で実施している。

1. ファイル名のハッシュ化
- `asset_category + logical_name + mime_type + payload` をもとに SHA-256 を取り、出力名に使う。
- 出力ファイル名は `assets/<sha256>.bin`。

2. 先頭バイト入れ替え
- 先頭 `16` バイトを逆順にして保存する。
- 復元時は同じ長さだけ再度逆順にして元へ戻す。

実装箇所:

- 先頭バイト入れ替え: [luminas_script.py](/home/web/luminous/LuminousScript/luminas_script.py#L389)
- ハッシュ名 + `.bin` 出力: [luminas_script.py](/home/web/luminous/LuminousScript/luminas_script.py#L398)

## Python側の生成フロー

### 1. 使用アセットの特定

CSVと `config.yml` から、以下の参照を走査する。

- 背景
- タイトル背景
- 立ち絵
- エフェクト
- BGM
- favicon

### 2. 画像最適化・形式決定

画像は既存ロジックを流用して必要に応じて縮小やWebP化を行う。
音声はそのまま読む。

### 3. 難読化済みファイルの書き出し

各アセットについて以下を行う。

- バイト列を得る
- 先頭16バイトを反転
- `assets/<hash>.bin` に保存
- HTMLに埋め込むためのメタ情報を返す

メタ情報の形式は概ね以下。

```json
{
  "path": "assets/<hash>.bin",
  "mimeType": "image/png",
  "headerSwapSize": 16
}
```

### 4. `file://` 直開き用補助ファイルの生成

`file://` では `fetch("assets/...")` がブラウザ制約で失敗しやすいため、各アセットごとに補助JSを出す。

- `assets/_local/<hash>.bin.js`

このJSは、難読化済みデータを Base64 文字列で `window.__LUMINA_LOCAL_ASSET_PACK__` に登録するだけの小さなスクリプト。

実装箇所: [luminas_script.py](/home/web/luminous/LuminousScript/luminas_script.py#L760)

## ブラウザ経路の分岐

### 1. `http://` / `https://` で開いた場合

通常配信時は以下の経路になる。

- `game.html`
- `fetch("assets/<hash>.bin")`
- `arrayBuffer()`
- 先頭16バイトを戻す
- Blob URL化
- `img.src` / `Audio.src` / CSS `background-image` に使用

この分岐は `location.protocol !== 'file:'` 側。

### 2. `file://` で直接開いた場合

直開き時は以下の経路になる。

- `game.html`
- 必要なアセットのときだけ `assets/_local/<hash>.bin.js` を `<script src>` で動的読込
- `window.__LUMINA_LOCAL_ASSET_PACK__[meta.path]` から Base64 を取得
- Base64 をバイト列に戻す
- 先頭16バイトを戻す
- Blob URL化

この分岐は `location.protocol === 'file:'` 側。

実装箇所: [luminas_script.py](/home/web/luminous/LuminousScript/luminas_script.py#L1853)

## 現状の制約

今回の構成は「HTML本体からはBase64を外し、アセットを外部化する」ことを優先している。
ただし `file://` 直開き対応のため、補助JS側には各アセットの Base64 を持たせている。

つまり現状は次の二重構造になっている。

- 通常配信用: `assets/<hash>.bin`
- `file://` 補助用: `assets/_local/<hash>.bin.js`

このため、直開き互換を残す限り、アセット実体は補助JS側にもう一度乗る。

## 補足

ブラウザで「読めるかどうか」と、生成時にファイルが「存在するかどうか」は別問題。

- `assets/*.bin` がある
  - Pythonが出力したことを意味する
- ブラウザが `file://` で読める
  - ブラウザのローカル読み込み制約に依存する

そのため、今回の実装では `http(s)` と `file://` で読み込み経路を分けている。
